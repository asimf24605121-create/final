🧠 1. TAB SYSTEM (CRITICAL FIX)
Issue:

Clicking “Password” still shows profile content.

Requirement:

Implement a tab-based system (no page reload).

Behavior:
View → shows profile data
Edit Profile → shows editable form
Password → shows password change form
HTML Structure:
<div class="tabs">
  <button class="tab-btn active" data-tab="view">View</button>
  <button class="tab-btn" data-tab="edit">Edit Profile</button>
  <button class="tab-btn" data-tab="password">Password</button>
</div>

<div id="view" class="tab-content active"></div>
<div id="edit" class="tab-content"></div>
<div id="password" class="tab-content"></div>
JavaScript:
const tabs = document.querySelectorAll(".tab-btn");
const sections = document.querySelectorAll(".tab-content");

tabs.forEach(tab => {
  tab.addEventListener("click", () => {
    tabs.forEach(t => t.classList.remove("active"));
    sections.forEach(s => s.classList.remove("active"));

    tab.classList.add("active");

    const target = tab.getAttribute("data-tab");
    document.getElementById(target).classList.add("active");
  });
});
📱 2. PHONE INPUT SYSTEM (STRICT VALIDATION)
Requirements:

Invalid numbers must be rejected
Use Library:
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/intl-tel-input@18/build/css/intlTelInput.css"/>
<script src="https://cdn.jsdelivr.net/npm/intl-tel-input@18/build/js/intlTelInput.min.js"></script>
Input:
<input type="tel" id="phone" required>
JS Setup:
const input = document.querySelector("#phone");

const iti = window.intlTelInput(input, {
  initialCountry: "auto",
  separateDialCode: true,
  preferredCountries: ["pk", "us", "gb"],
  utilsScript: "https://cdn.jsdelivr.net/npm/intl-tel-input@18/build/js/utils.js"
});
Validation:
function validatePhone() {
  if (!iti.isValidNumber()) {
    alert("Invalid phone number");
    return false;
  }
  return true;
}
Save Format:
const phone = iti.getNumber(); // Example: +923001234567
